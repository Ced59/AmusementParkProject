# P09.5 — Révision des utilitaires et pipes transverses

## Objectif

Nettoyer les petits helpers transverses pour éviter :

- la duplication de logique de localisation ;
- la dispersion des helpers Angular techniques ;
- les imports historiques depuis des zones legacy quand un emplacement `shared` plus propre existe.

## Décisions prises

### Localisation transverse

Un point d'entrée officiel est introduit dans :

```text
src/app/shared/utils/localization/
```

Il regroupe désormais :

- `resolveLocalizedValue`
- `resolveLocalizedText`
- `stripHtml`
- `isRichTextEmpty`

Le pipe transverse `localizedText` est aligné sur ce même socle pour éviter toute double implémentation.

### Pipe transverse

Le pipe officiel vit dans :

```text
src/app/shared/pipes/localized-text.pipe.ts
```

L'ancien fichier `src/app/pipes/localized-text.pipe.ts` n'est plus conservé.

### Helper Angular technique

Le helper `commitViewUpdate` est promu dans :

```text
src/app/shared/utils/angular/
```

L'ancien fichier `src/app/utils/change-detection.utils.ts` n'est plus conservé.

### Cible retenue

La phase P09.5 est appliquée directement sur la cible finale :

- pas de wrapper de compatibilité ;
- pas de double point d'entrée legacy/shared ;
- imports convergés directement vers `shared`.

## Règles à appliquer à partir de P09.5

- toute nouvelle logique transverse de localisation doit partir de `shared/utils/localization` ;
- les nouveaux pipes transverses doivent vivre dans `shared/pipes` ;
- les petits helpers Angular génériques réutilisables entre plusieurs domaines doivent converger vers `shared/utils` ;
- les anciens chemins legacy ne doivent plus être réintroduits.
