# Front components → features migration

Cette passe déplace les anciens composants `src/app/components` vers des emplacements feature-first ou shared-first.

## Règles appliquées

- pages publiques → `src/app/features/public/<feature>/pages`;
- pages admin → `src/app/features/admin/<feature>/pages`;
- composants admin réutilisables → `src/app/features/admin/shared/ui`;
- composants transverses → `src/app/shared/components`;
- composants auth/profile → `src/app/features/auth` et `src/app/features/profile`;
- `ChangeDetectionStrategy.OnPush` ajouté aux composants legacy migrés qui ne l’avaient pas encore.
