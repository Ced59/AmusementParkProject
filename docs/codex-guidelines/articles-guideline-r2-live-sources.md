# AmusementPark — Guideline éditoriale Articles

Date : **2026-06-30**  
Version : **r2 — sources vérifiées et style public naturel**  
Projet : **amusement-parks.fun**

Ce document donne à Codex les règles à appliquer pour les articles de parc, d’attraction, d’histoire, de visite, de médias et d’actualité durable.

---

## 1. Rôle des articles

Les articles ne remplacent pas les descriptions des parcs, zones ou parkItems. Ils servent à développer un sujet qui mérite plus de contexte.

Un article peut raconter :

- l’histoire d’un parc ;
- l’histoire d’une attraction ;
- une ouverture importante ;
- une fermeture définitive ;
- une relocalisation ;
- un changement d’opérateur ;
- une visite terrain ;
- une captation photo ou vidéo ;
- une page de patrimoine de loisirs ;
- une actualité durable.

Un article ne doit pas devenir une fiche technique déguisée.

---

## 2. Quand créer un article

Créer un article seulement si le sujet apporte une vraie valeur éditoriale.

Bons sujets :

- parc majeur ;
- attraction emblématique ;
- événement historique ;
- nouveauté importante ;
- fermeture ou transformation marquante ;
- constructeur ou exploitant majeur ;
- visite complète ;
- vidéo onride ou offride originale ;
- comparaison historique documentée.

Mauvais sujets :

- micro-changement sans intérêt durable ;
- article qui répète la description ;
- texte sans source ;
- contenu promotionnel vide ;
- fiche technique transformée en article ;
- sujet artificiel créé pour remplir le site.

---

## 3. Sources

Un article doit s’appuyer sur des sources fiables.

Sources possibles :

- site officiel du parc ;
- communiqué officiel ;
- presse locale ou nationale ;
- archives de presse ;
- documents historiques ;
- bases spécialisées fiables ;
- vidéos ou photos originales ;
- documents administratifs publics.

Les faits importants doivent être vérifiés : dates, ouverture, fermeture, opérateur, constructeur, relocalisation, changement majeur, chiffres et déclarations officielles.

Pour les informations récentes ou susceptibles d’avoir changé, vérifier les sources actuelles avant de rédiger.

---

## 4. Style

Le style doit être :

- naturel ;
- clair ;
- agréable à lire ;
- documenté ;
- orienté lecteur ;
- non promotionnel ;
- non mécanique ;
- non académique.

L’article doit raconter quelque chose et donner envie d’aller au bout.

---

## 5. Structure recommandée

Structure possible :

1. Introduction courte.
2. Contexte du parc, de l’attraction ou de l’événement.
3. Développement chronologique ou thématique.
4. Informations vérifiées et sources.
5. Impact sur l’histoire, la visite ou la page du parc.
6. Conclusion naturelle.

La structure peut être adaptée au sujet, mais la progression doit rester lisible.

---

## 6. Articles historiques

Pour un article historique, privilégier :

- chronologie claire ;
- fondateurs ;
- exploitants successifs ;
- attractions importantes ;
- périodes de développement ;
- transformation ou fermeture si sourcée ;
- traces actuelles ;
- rôle dans le patrimoine local ou l’industrie des loisirs.

Le texte peut être vivant, mais les faits doivent rester vérifiés.

---

## 7. Articles de visite et vidéos

Pour une visite terrain :

- préciser la date ;
- distinguer les faits des ressentis ;
- mentionner les observations réelles ;
- rattacher les photos ou vidéos utiles ;
- ne pas transformer une observation ponctuelle en règle générale.

Pour une vidéo onride ou offride :

- préciser le parc ;
- préciser l’attraction ;
- préciser le type de captation ;
- donner le contexte de l’attraction ;
- éviter de surcharger avec des données techniques ;
- renvoyer vers les pages utiles du site.

---

## 8. Multilingue

Langues possibles :

- français ;
- anglais ;
- espagnol ;
- allemand ;
- italien ;
- néerlandais ;
- portugais ;
- polonais.

La traduction doit être naturelle. Les titres doivent être adaptés à chaque langue, pas copiés mot à mot.

---

## 9. Titres, méta et liens internes

Un titre doit être clair, spécifique et humain.

Éviter :

- titre générique ;
- titre clickbait ;
- promesse non tenue ;
- sur-optimisation SEO ;
- répétition artificielle de mots-clés.

La meta description doit résumer le sujet et donner envie de lire.

Les liens internes doivent aider le lecteur : parc, attraction, constructeur, opérateur, fondateur, vidéo, galerie ou article lié.

---

## 10. Relation avec les descriptions

Les descriptions donnent une présentation courte ou moyenne intégrée aux fiches.

Les articles développent :

- histoire ;
- contexte ;
- chronologie ;
- visite ;
- média ;
- patrimoine ;
- actualité durable.

Ne pas dupliquer mot pour mot une description dans un article.

---

## 11. Checklist avant livraison

Avant de livrer un article, vérifier :

- le sujet mérite un article ;
- les faits importants sont sourcés ;
- le texte est naturel ;
- le titre est spécifique ;
- la structure est lisible ;
- l’article n’est pas une fiche technique ;
- l’article ne copie pas une description ;
- les liens internes sont utiles ;
- les dates sont vérifiées ;
- les incertitudes sont signalées ;
- les traductions sont naturelles si fournies.

---

## 12. Résumé pour Codex

Codex doit créer ou modifier un article seulement lorsqu’un vrai angle éditorial existe. L’article doit raconter, documenter et contextualiser. Il doit être sourcé, naturel, utile et durable, sans remplacer la description ni se transformer en fiche technique.
