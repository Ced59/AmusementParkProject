Roadmap Admin · Fast Edition

ADMIN
ULTRA FAST

Roadmap responsive pour transformer l’administration AmusementPark en poste de production : ajout manuel extrêmement rapide, import assisté, graph upsert visuel, édition inline, mobile réel, audit et qualité éditoriale. Le principe directeur est simple : accélérer massivement sans casser l’architecture actuelle.

7phases
56sous-tâches
1règle absolue
0architecture parallèle

Règle absolue et générale

Toute évolution admin doit respecter à fond l’architecture actuelle, les principes SOLID, la séparation des responsabilités, les facades de state, les services data-access, les contrats API propres, les handlers applicatifs et les composants partagés lorsque leur réutilisation est pertinente. La vitesse de saisie ne doit jamais justifier un contournement sale du domaine ou une duplication durable.

Architecture existanteOn étend les features actuelles au lieu de créer un second système admin.
SOLIDParsing, mapping, validation, state et UI restent séparés et testables.
Réutilisation pertinenteOn mutualise les composants utiles, sans forcer une abstraction artificielle.
Sécurité éditorialePreview avant apply, audit des actions, ToReview par défaut pour les flux massifs.

M25 → M27 : accélération manuelle immédiate

Socle propre, workbench par parc, création rapide, pagination, inline edit et bulk métier.

M28 → M29 : import puissant mais contrôlé

Collage/CSV, prévisualisation, dédoublonnage, Graph Upsert visuel et historique exploitable.

M30 → M31 : qualité, mobile et exploitation

Complétude éditoriale, localisation, gates de publication, responsive mobile réel, tests et observabilité.
